## 1. 부분 피봇팅(Partial Pivoting)

### 1.1 단순 가우스 소거법의 문제점

단순 가우스 소거법에서는 각 소거 단계에서 피봇 요소 $a_{ii}$가 직접 나눗수로 사용됩니다. 이로 인해 두 가지 치명적인 문제가 발생합니다:

1. **0으로 나눗셈**: $a_{ii} = 0$이면 알고리즘이 즉시 실패
2. **큰 반올림 오차**: $a_{ii}$가 매우 작으면(0에 가까우면), 승수 $a_{ki}/a_{ii}$가 매우 커져 후속 연산에서 모든 반올림 오차를 해당 배수만큼 증폭

두 문제 모두 동일한 근본 원인에서 비롯됩니다: 피봇 요소가 해당 열의 다른 요소에 비해 너무 작습니다.

### 1.2 부분 피봇팅 전략

해결책은 간단합니다: 열 $i$에 대한 소거를 수행하기 전에, $i$행부터 $n$행까지에서 열 $i$의 **최대 절대값**을 가진 행을 찾아 해당 행과 $i$행을 교환합니다. 이렇게 하면 피봇이 항상 해당 열에서 가장 큰 가용 요소가 되어 승수의 크기를 최소화합니다.

승수가 $\text{factor} = a_{ki}/a_{ii}$로 계산되므로, $|a_{ii}|$를 가능한 한 크게 만들면 모든 계수가 $|\text{factor}| \leq 1$을 만족하게 됩니다. 이는 각 단계에서 오차 증폭을 제한합니다.

### 1.3 수정된 알고리즘 의사코드

단순 가우스 소거법에 대한 유일한 수정은 각 소거 단계 전에 피봇 탐색과 행 교환을 추가하는 것입니다:

```
for i = 1 to n-1:
    // Find max |a[k][i]| for k = i, ..., n
    imax = argmax(|a[k][i]|) for k = i..n
    swap row i with row imax
    // Proceed with standard elimination
    for k = i+1 to n:
        factor = a[k][i] / a[i][i]
        for j = i to n+1:
            a[k][j] = a[k][j] - factor * a[i][j]
```

> **참고:** 행 교환은 소거 단계당 $O(n)$ 연산이므로, 피봇팅의 총 오버헤드는 $O(n^2)$입니다 — 이는 소거 자체의 $O(n^3)$ 비용에 비해 무시할 수 있습니다. 피봇팅을 생략할 이유는 본질적으로 없습니다.

## 2. 스케일링된 부분 피봇팅(Scaled Partial Pivoting)

때로는 부분 피봇팅만으로는 불충분합니다. 이는 행렬의 행들이 **매우 다른 크기**를 가질 때 발생합니다. 모든 성분이 큰 행이 자체 스케일에 비해 최선의 선택이 아닌데도 피봇 선택을 지배할 수 있습니다.

**스케일링된 부분 피봇팅(Scaled Partial Pivoting)**은 피봇 후보를 비교하기 전에 각 행을 정규화하여 이를 해결합니다:

1. 각 행의 **스케일 팩터(scale factor)**를 계산: $s_i = \max_j |a_{ij}|$
2. 열 $i$에 대한 피봇을 선택할 때, $|a_{ki}| / s_k$ (후보 피봇과 행의 스케일 팩터의 비율)를 최대화하는 행 $k$를 선택
3. 교환 후 표준 소거 진행

스케일 팩터 $s_i$는 처음에 한 번 계산되고 행이 교환될 때만 갱신됩니다(스케일 팩터는 해당 행을 따라갑니다). 이 정규화는 피봇 선택이 각 행의 전체 크기를 고려하도록 하여, 균일하게 큰 성분을 가진 행이 항상 선택되는 것을 방지합니다.

> **[선형대수]** 스케일링된 부분 피봇팅은 부분 피봇팅을 적용하기 전에 행렬의 행을 암묵적으로 스케일링하는 것과 동등합니다. 핵심 통찰은 "최선의" 피봇이 반드시 절대값이 가장 큰 것이 아니라, 해당 행의 다른 성분에 비해 가장 큰 것이라는 점입니다.

## 3. 피봇팅 필요성을 보여주는 예제

다음 시스템을 고려합니다:

$$0.0003x_1 + 3.0000x_2 = 2.0001$$

$$1.0000x_1 + 1.0000x_2 = 1.0000$$

정확한 해는 $x_1 = 1/3$, $x_2 = 2/3$입니다.

### 3.1 피봇팅 없이 (4자리 산술)

주어진 순서대로 4자리 산술을 사용:

- 피봇은 $a_{11} = 0.0003$ (매우 작음!)
- 계수 = $1.0000 / 0.0003 = 3333.3$ (매우 큼!)
- 소거 후: $-9999x_2 = -6666$
- 후진 대입: $x_2 = 0.6667$, 그 다음 $x_1 = (2.0001 - 3.0000 \times 0.6667) / 0.0003 = 0.3333$

제한된 정밀도에서 답은 **부정확합니다**. 3333.3이라는 거대한 계수가 모든 반올림 오차를 그만큼 증폭합니다. 4자리 반올림으로 도입된 작은 섭동이 유효 자릿수를 잃을 정도로 확대됩니다.

### 3.2 피봇팅 적용

먼저 두 행을 교환합니다 (1열에서 $|1.0000| > |0.0003|$이므로):

$$1.0000x_1 + 1.0000x_2 = 1.0000$$

$$0.0003x_1 + 3.0000x_2 = 2.0001$$

이제:

- 피봇은 $a_{11} = 1.0000$ (적절한 스케일)
- 계수 = $0.0003 / 1.0000 = 0.0003$ (작은 계수!)
- 소거 후: $2.9997x_2 = 1.9998$
- 후진 대입: $x_2 = 0.6667$, 그 다음 $x_1 = (1.0000 - 1.0000 \times 0.6667) / 1.0000 = 0.3333$

작은 계수(3333.3 대신 0.0003)가 반올림 오차를 증폭하지 않으므로 결과가 훨씬 정확합니다.

> **참고:** 이 예제는 피봇팅이 왜 중요한지 극적으로 보여줍니다. 3333.3이라는 계수가 모든 반올림 오차를 그만큼 증폭합니다. 피봇팅은 계수를 $\leq 1$로 유지하여 오차 성장을 제어합니다. 실무에서는 항상 부분 피봇팅이 사용됩니다 — 단순 가우스 소거법은 교육 목적으로만 가르칩니다.

## 4. 가우스-조르단 방법(Gauss-Jordan Method)

**가우스-조르단 방법(Gauss-Jordan method)**은 표준 가우스 소거법을 확장하여 대각선 **위와 아래** 모두의 성분을 소거함으로써 상삼각 행렬 대신 **대각 행렬(diagonal matrix)**을 생성합니다.

절차는 다음과 같습니다:

1. **전진 소거**: 표준 가우스 소거법과 동일 — 대각선 아래를 소거
2. **후진 소거**: 추가로 대각선 **위**의 성분도 소거
3. **정규화**: 각 행을 대각 요소로 나누어 좌측에 단위 행렬을 생성

결과적으로 확대 행렬 $[A|b]$가 $[I|x]$로 변환되며, 여기서 $x$가 해입니다. 행렬이 이미 대각(또는 정규화 후 단위)이므로 후진 대입이 필요하지 않습니다.

**주요 특성:**

- **후진 대입 불필요**: 변환된 우변에서 직접 해를 읽을 수 있음
- **역행렬 계산에 유용**: $[A|I]$에 가우스-조르단을 적용하면 $[I|A^{-1}]$을 얻음
- **더 비쌈**: 연산 횟수는 $O(n^3)$이지만 표준 가우스 소거법보다 큰 상수를 가짐 — 후진 대입 포함 표준 가우스의 $n^3/3$ 곱셈에 비해 약 $n^3/2$ 곱셈

> **[선형대수]** 가우스-조르단 방법은 역행렬 계산의 표준 알고리즘입니다. 단일 시스템을 풀 때는 후진 대입을 포함한 가우스 소거법보다 약 50% 더 비싸지만, 여러 우변을 동시에 풀 때(역행렬 계산 등)는 경쟁력이 있습니다.

## 5. 삼중대각 시스템 — 토마스 알고리즘(Thomas Algorithm)

### 5.1 삼중대각 행렬 구조

**삼중대각 행렬(tridiagonal matrix)**은 주대각선, 바로 위의 대각선(상대각선, super-diagonal), 바로 아래의 대각선(하대각선, sub-diagonal)에만 0이 아닌 성분이 있습니다. 나머지 모든 성분은 0입니다:

$$\begin{bmatrix} f_1 & g_1 & & \\ e_2 & f_2 & g_2 & \\ & \ddots & \ddots & \ddots \\ & & e_n & f_n \end{bmatrix} \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_n \end{bmatrix} = \begin{bmatrix} r_1 \\ r_2 \\ \vdots \\ r_n \end{bmatrix}$$

여기서 $e_i$는 하대각선, $f_i$는 주대각선, $g_i$는 상대각선입니다.

삼중대각 시스템은 과학 계산에서 매우 흔합니다:

- **이산화된 상미분방정식(Discretized ODEs)**: 2차 경계값 문제에 대한 중심 유한 차분
- **스플라인 보간(Spline interpolation)**: 3차 스플라인 계수 시스템
- **음해법(Implicit time-stepping)**: 포물선 편미분방정식(예: 열 방정식)에 대한 음해법

### 5.2 토마스 알고리즘

**토마스 알고리즘(Thomas Algorithm)**은 삼중대각 구조를 활용하는 가우스 소거법의 특수 형태입니다. 두 단계로 구성됩니다:

**전진 소거(Forward sweep)** (소거): $i = 2$부터 $n$까지:

$$\text{factor} = \frac{e_i}{f_{i-1}}$$

$$f_i = f_i - \text{factor} \cdot g_{i-1}$$

$$r_i = r_i - \text{factor} \cdot r_{i-1}$$

**후진 대입(Back substitution)**:

$$x_n = \frac{r_n}{f_n}$$

$i = n-1$부터 $1$까지:

$$x_i = \frac{r_i - g_i \cdot x_{i+1}}{f_i}$$

전진 소거의 각 단계는 현재 행과 바로 위의 행만 접근하고, 후진 대입의 각 단계는 다음 해 값만 필요합니다. 알고리즘은 세 개의 대각선만 접근하며 밴드 외부의 성분을 채우지 않습니다.

### 5.3 계산 비용

토마스 알고리즘은 전진 소거에서 정확히 $3(n-1)$회의 곱셈/나눗셈과 $3(n-1)$회의 덧셈/뺄셈, 후진 대입에서 $n$회의 연산이 필요합니다 — 총 비용은 **$O(n)$**입니다.

이는 일반 가우스 소거법의 $O(n^3)$ 비용에 비해 극적인 개선입니다. 크기 $n = 100{,}000$인 시스템에서 토마스 알고리즘은 약 $10^{10}$배 더 빠릅니다.

> **[자료구조]** 토마스 알고리즘은 행렬의 밴드 구조를 활용합니다. 3개의 대각선만 저장하고 처리하며, 이는 희소 행렬 형식이 메모리를 절약하는 것과 유사합니다. 이는 반복되는 주제입니다: 문제의 구조를 활용하면 극적인 효율성 향상으로 이어집니다.

## 6. Python 구현

Python의 NumPy/SciPy 라이브러리는 내부적으로 부분 피봇팅을 포함한 LAPACK 루틴을 사용하는 견고하고 실용적 수준의 구현을 제공합니다:

```python
import numpy as np

x = np.linalg.solve(A, b)  # Uses LAPACK with pivoting internally
```

삼중대각 시스템의 경우 SciPy가 특화된 솔버를 제공합니다:

```python
from scipy.linalg import solve_banded

# ab stores the diagonals in banded format
x = solve_banded((1, 1), ab, b)  # Exploits tridiagonal structure
```

> **참고:** 실무 코드에서는 가우스 소거법을 직접 구현하는 대신 항상 `np.linalg.solve()` 또는 `scipy.linalg.solve()`를 사용하십시오. 이 함수들은 부분 피봇팅, 캐시 효율적 블로킹, 오류 처리를 포함하는 고도로 최적화된 LAPACK 루틴(DGESV)을 사용합니다.

## 요약

| 방법 | 피봇팅? | 비용 | 최적 용도 |
|:-----|:--------|:-----|:----------|
| 단순 가우스(Naive Gauss) | 없음 | $O(n^3)$ | 교육용으로만 |
| 부분 피봇팅 가우스(GEPP) | 있음 | $O(n^3)$ | 일반 시스템 |
| 가우스-조르단(Gauss-Jordan) | 있음 | $O(n^3)$ (더 큰 상수) | 역행렬 계산 |
| 토마스 알고리즘(Thomas Algorithm) | 없음 | $O(n)$ | 삼중대각 시스템 |

핵심 요점:

- **일반 시스템에는 항상 피봇팅을 사용** — 비용은 무시할 수 있으며 안정성 향상은 필수적
- **가능할 때마다 구조를 활용** — 토마스 알고리즘은 삼중대각 시스템에서 $O(n^3)$ 문제를 $O(n)$으로 전환
- **실무에서는 라이브러리 구현을 사용** — 최적화되고 테스트되었으며 예외 상황을 처리
